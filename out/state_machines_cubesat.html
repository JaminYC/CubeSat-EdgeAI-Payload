01Introduccion: 3 niveles de jerarquia

Una maquina de estados de CubeSat tiene 3 capas. Cuando el satelite esta en un modo de mision, internamente cada subsistema esta en su propio estado.

La regla: cada subsistema tiene su FSM propia. La FSM de mision NO controla detalles; solo dice "estamos en NOMINAL". Cada subsistema interpreta NOMINAL en su contexto local.

Como leerla: cuando el OBC dice "modo NOMINAL", todos los subsistemas saben que tienen que estar en su estado correspondiente. Si EPS detecta BATTERY_CRITICAL, fuerza al satelite entero a SAFE.

04OBC — On-Board Computer

El "cerebro central" del satelite. Coordina todos los subsistemas.

stateDiagram-v2 [*] --> BOOT BOOT --> NOMINAL: init complete BOOT --> FAULT: init fail NOMINAL --> MAINTENANCE: scheduled (eclipse window) MAINTENANCE --> NOMINAL: tasks done NOMINAL --> FAULT: watchdog timeout / SEU NOMINAL --> SAFE_HOLD: ground CMD_SAFE_OBC SAFE_HOLD --> NOMINAL: ground CMD_RESUME FAULT --> NOMINAL: auto-reboot OK FAULT --> SAFE_HOLD: 3 reboots failed

05EPS — Sistema de potencia

El corazon energetico. Si EPS entra en BATTERY_CRITICAL, fuerza SAFE en todo el satelite.

stateDiagram-v2 [*] --> BOOT BOOT --> BATTERY_NOMINAL: SoC > 70% BOOT --> BATTERY_LOW: 30% < SoC < 70% BOOT --> BATTERY_CRITICAL: SoC < 30% BATTERY_NOMINAL --> CHARGING: sun & SoC < 100% BATTERY_NOMINAL --> DISCHARGING: eclipse CHARGING --> BATTERY_NOMINAL: SoC > 90% DISCHARGING --> BATTERY_LOW: SoC < 70% BATTERY_LOW --> BATTERY_NOMINAL: SoC > 80% BATTERY_LOW --> BATTERY_CRITICAL: SoC < 30% BATTERY_CRITICAL --> BATTERY_LOW: SoC > 40% BATTERY_CRITICAL --> FAULT: SoC < 10% FAULT --> [*]: BMS shutdown (irrecoverable) note right of BATTERY_CRITICAL fuerza el bus a SAFE, apaga rails no esenciales end note

06ADCS — Control de actitud

Controla hacia donde apunta el satelite. El ADCS tipico tiene 5 modos principales.

stateDiagram-v2 [*] --> OFF OFF --> DETUMBLE: post-deployment DETUMBLE --> SUN_ACQUISITION: rates < 1 deg/s SUN_ACQUISITION --> SUN_POINTING: sun vector found SUN_POINTING --> NADIR_POINTING: CMD nadir SUN_POINTING --> TARGET_POINTING: CMD target (lat,lon) NADIR_POINTING --> SUN_POINTING: CMD sun TARGET_POINTING --> SUN_POINTING: target done SUN_POINTING --> INERTIAL_HOLD: eclipse start INERTIAL_HOLD --> SUN_POINTING: eclipse end SUN_POINTING --> SAFE: gyro fault NADIR_POINTING --> SAFE: gyro fault TARGET_POINTING --> SAFE: gyro fault SAFE --> DETUMBLE: ground CMD_RESUME

07COMMS — Comunicaciones

Manejo del radio (UHF para uplink/downlink basico, S-Band para datos).

stateDiagram-v2 [*] --> IDLE IDLE --> BEACON: timer (cada 30s) BEACON --> IDLE: beacon sent IDLE --> RX: carrier detected RX --> COMMAND_PROCESSING: valid frame COMMAND_PROCESSING --> TX_ACK: cmd valid COMMAND_PROCESSING --> RX: cmd invalid (drop) TX_ACK --> IDLE: ack sent IDLE --> TX: CMD_DOWNLINK (telemetria queued) TX --> IDLE: queue empty IDLE --> FAULT: radio temp > 85 C FAULT --> IDLE: temp normalized

10Referencias para validar

CubeSats con FSM publicada que podes mirar como benchmark.

  • OreSat (PSU) — CubeSat 100% open con FSM completa en Python.
    github.com/oresat
    El mas cercano a tu caso: academico, RPi/MCUs, comms por CAN.
  • UPSat (Libre Space Foundation) — Primer CubeSat 100% open en orbita (2017).
    gitlab.com/librespacefoundation/upsat
  • AAUSAT-3/4 (Aalborg) — La documentacion academica mas detallada del mundo.
    space.aau.dk
    Buscar "AAUSAT-3 Operations Concept" — 50+ paginas.
  • NASA cFS Limit Checker — Plantilla de referencia para FDIR.
    github.com/nasa/lc
  • NASA F' (FPrime) — Genera FSM ejecutable desde XML. Usado en MarCO e Ingenuity.
    nasa.github.io/fprime
  • ECSS-E-ST-70-11C — Estandar ESA para "Space segment operability".
    ecss.nl/ecss-e-st-70-11c
  • OPS-SAT operational concept — Satelite ESA experimental con docs publicos.
    esa.int/OPS-SAT
